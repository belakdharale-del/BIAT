You are a senior Python data science engineer, Streamlit developer, and UI/UX designer.

BIAT Risk Monitor

============================================================
1. PROJECT CONTEXT
============================================================

The project is about predicting the risk of overdraft limit breach for BIAT bank clients.

The system analyzes historical monthly banking data and predicts whether a client is likely to exceed their overdraft limit 30 days in advance.

The final system must include:

1. Data preparation pipeline
2. Feature engineering pipeline
3. Machine learning model training
4. LightGBM scoring
5. Risk evolution analysis
6. Operational clients-to-notify list
7. Professional Streamlit web application
8. Model performance page
9. Rule-based chatbot/assistant based on CSV data
10. Clean project architecture

The project must be in French for the user interface because it is intended for a Tunisian/French-speaking banking context.

============================================================
2. BUSINESS OBJECTIVE
============================================================

The objective is to help BIAT account managers, risk analysts, and managers:

- Identify clients likely to exceed their overdraft limit within 30 days.
- Assign each client a risk score from 0% to 100%.
- Convert the score into business alert levels.
- Prioritize clients that need action.
- Monitor worsening and improvement of client behavior over time.
- Display global KPIs for managers.
- Display individual client profiles for account managers.
- Export lists of clients to contact.
- Ask questions to a chatbot-like assistant based on the portfolio data.

The application should be presented as a real banking decision-support system.

6. DATA PREPARATION SCRIPT
============================================================

Create:

src/1_data_preparation.py

This script must:

1. Load:
   data/f1020232025_3.xlsx

2. Normalize column names:
   - strip spaces
   - replace spaces with underscores
   - replace hyphens with underscores

3. Detect essential columns automatically:
   - PERIODE
   - CPTE
   - DATE_OF_BIRTH if available

4. Convert:
   - PERIODE to datetime
   - DATE_OF_BIRTH to datetime if available

5. Drop rows where:
   - PERIODE is missing
   - CPTE is missing

6. Fill missing financial values with zero:
   - NBRJDEP
   - NBRJDEB
   - RISK_BRUT
   - MVT
   - NET_MONTHLY

7. Convert financial columns to numeric.

8. Cap NBRJDEP:
   - minimum 0
   - maximum 31

Because one month cannot have more than 31 days.

9. Calculate AGE:
   AGE = difference between PERIODE and DATE_OF_BIRTH in years.
   Clip AGE between 18 and 80.
   If DATE_OF_BIRTH is unavailable, create AGE = 40.

10. Fill missing categorical columns:
   - GENDER
   - SECTOR
   - MARITAL_STATUS
   - EMPLOYMENT_STATUS
   - POST_CODE

Use "Unknown" when missing.

11. Sort data by:
   CPTE, PERIODE

12. Save:
   outputs/prepared_data.csv

The script must print clear diagnostics:
- initial rows
- final rows
- number of unique accounts
- min period
- max period
- missing values
- output file path

Create:

src/2_feature_engineering.py

Input:
outputs/prepared_data.csv

Output:
outputs/modeling_data.csv

This script must create the target variable and behavioral features.

TARGET VARIABLE:

The model predicts whether the client will be risky next month.

Create:

NBRJDEP_NEXT = groupby("CPTE")["NBRJDEP"].shift(-1)
RISK_BRUT_NEXT = groupby("CPTE")["RISK_BRUT"].shift(-1)

Then:

target = 1 if:
NBRJDEP_NEXT > 3 OR RISK_BRUT_NEXT > 0

else:
target = 0

Important:
Remove the last observation of each client because it has no next month.
Then drop:
- NBRJDEP_NEXT
- RISK_BRUT_NEXT

This avoids data leakage.

FEATURES TO CREATE:

1. AVG_DEP_3M
Average NBRJDEP over the previous 3 months.
Use shift(1) before rolling.

2. DAYS_OVD_6M
Total NBRJDEP over the previous 6 months.
Use shift(1) before rolling.

3. REV_VARIANCE
Standard deviation of MVT over the previous 3 months.
Use shift(1) before rolling.

4. NBRJDEP_LAG1
Previous month NBRJDEP.

5. DELTA_NBRJDEP
Current NBRJDEP - previous month NBRJDEP.

6. MOIS_CONSEC
Number of consecutive months where NBRJDEP > 0.

7. Keep AGE from the prepared data.

Final columns should include:

- target
